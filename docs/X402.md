# x402 / USDC Settlement & Governance Hardening

This adds the **x402 agentic-payment rail** (Coinbase + Cloudflare's HTTP-402
standard, settled in USDC) as a first-class payment provider, plus three
governance upgrades that make the rail safe to run: graduated spend permissions,
settlement-fee-aware profitability, and value-aware audit thresholds.

## 1. x402 / USDC payments

`X402PaymentService` (`sovereign_os/payments/x402.py`) implements the same
`PaymentService.charge()` contract as Stripe/Dummy, so the job queue, ledger, and
webhooks are unchanged. It settles a job's revenue in USDC to a payout address
and returns a settlement reference (tx hash in live mode, deterministic
pseudo-hash in sandbox).

**Two modes**

| Mode | When | Behavior |
|---|---|---|
| **Sandbox** (default) | `X402_SANDBOX=true` or no facilitator URL | No network calls. Deterministic, idempotent settlement id derived from the charge inputs. Never moves real funds. Safe for CI and testnet rehearsal. |
| **Live** | `X402_SANDBOX=false` **and** `X402_FACILITATOR_URL` set | POSTs `{network, asset, payTo, amount, ...}` to `<facilitator>/settle` with an idempotency key; returns the tx hash. 3 retries with backoff. Requires `requests`. |

**Selection** (`create_payment_service()`):
1. `PAYMENT_PROVIDER=x402|stripe|dummy` forces the provider.
2. Otherwise x402 is auto-selected when `X402_PAY_TO` is set; else Stripe if
   `STRIPE_API_KEY` is set; else Dummy.

**Config** (all optional; testnet defaults): `X402_PAY_TO`, `X402_NETWORK`
(default `base-sepolia`), `X402_ASSET` (default `USDC`), `X402_SANDBOX`
(default `true`), `X402_FACILITATOR_URL`, `X402_API_KEY`. See `.env.example`.

USDC has 6 decimals: 1 cent = 10,000 atomic units (`cents_to_usdc_atomic`).

## 2. Graduated spend permissions

`SovereignAuth` still gates `SPEND_USD` by TrustScore (≥80), but autonomous spend
is now **graduated**: once an agent clears the threshold, its per-task spend
ceiling scales linearly with trust.

- `max_spend_cents_for(agent_id)` — 0 below threshold; `autonomous_spend_min_cents`
  at the threshold; `autonomous_spend_max_cents` at TrustScore 100.
- `can_spend(agent_id, amount_cents)` — gate + ceiling in one check.
- `record_audit(agent_id, passed=, score=)` — trust delta scaled by audit
  quality (a strong pass earns more than a marginal one).
- `persist_path=` — TrustScore + audit history survive restarts.

## 3. Settlement-fee-aware profitability (CFO)

`FiscalBoundaries.settlement_fee_ratio` (0–1, default 0) nets job revenue down by
expected settlement cost (x402 facilitator/network fees, Stripe fees) **before**
the margin check, so `Treasury.approve_job_profitability` reasons about funds
that actually land. With the margin floor disabled, a fee alone can still reject
a job that would settle at a loss.

## 4. Runway-floor guard (CFO)

`FiscalBoundaries.runway_floor_days` (default 0 = off). `Treasury.approve_task`
projects runway from the **actual** trailing-7-day burn rate
(`projected_runway_days`) and denies a task if approving it would push projected
runway below the floor.

## 5. Value-aware audit thresholds (Auditor)

`value_aware_min_score(job_revenue_cents)` raises the passing bar with the payout
(+0.1 per $50, capped at 0.9). `ReviewEngine.audit_task(..., min_score=)` enforces
it: higher-paid deliverables must clear a stricter score, and the Judge LLM's
lenient `passed` flag can't override it. Cheap/free work keeps the default
leniency (`min_score=None`).

All five are backward compatible — every new charter field defaults to a value
that preserves existing behavior, and the existing test suite passes unchanged.
